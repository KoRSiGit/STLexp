# ООО «Сибтехлит» — Официальный веб-сайт и ИИ-Ассистент

Современный, быстрый и отзывчивый веб-сайт для компании **«Сибтехлит»** (Сибирские Технологии Литья, г. Новосибирск). Сайт разработан на основе стека **React 19**, **Vite**, **TypeScript**, **Tailwind CSS v4** и включает встроенный модуль интеллектуальных расчетов параметров плавок/смесей, а также полноценного интерактивного **ИИ-Инженера**, использующего современные модели Gemini.

---

## 🛠️ Стек технологий

*   **Frontend:** React 19, TypeScript, Tailwind CSS v4 (с плагином `@tailwindcss/vite`).
*   **Backend:** Node.js, Express (служит для безопасного проксирования запросов к API и раздачи статики).
*   **ИИ-Ассистент:** `@google/genai` (Gemini API).
*   **Анимации:** `motion/react` (плавные интерактивные переходы и анимация табов).
*   **Иконки:** `lucide-react`.

---

## ⚡️ Автономный и Статический режимы (Для GitHub Pages)

Данное приложение поддерживает **комбинированную гибридную архитектуру**. Если бэкенд-сервер недоступен (например, при размещении на статической платформе вроде **GitHub Pages**), все динамические функции плавно переключаются в локальный автономный режим работы:
1. **ИИ-Ассистент (Оффлайн режим):** При блокировке запросов к бэкенду чат с ИИ переключается на встроенный экспертный симулятор металлургических справочных данных. Он реагирует на ключевые литейные темы (ХТС-процесс, смола, регенерация, индукционные печи, плавка) и предлагает корректное инженерное руководство по режимам технологических операций СЛТ.
2. **Отправка спецификаций КП:** При нажатии «Отправить запрос» в корзине оборудования система автоматически создаст уникальную спецификацию с идентификатором `КД-YYMMDD-XXXX-STATIC` в памяти браузера, выведет подтверждение успешной сессии и предложит контакты для быстрого прямого запроса комплекта на `info@sibtehlit.ru`.

## 🚀 Пошаговое руководство по сборке и локальному запуску

Для работы проекта требуется установленная среда выполнения **Node.js** (версии 18 или новее).

### 1. Клонирование репозитория и установка зависимостей
Склонируйте репозиторий с GitHub и установите все необходимые пакеты:
```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ_НА_GITHUB>
cd sibtehlit-site
npm install
```

### 2. Настройка переменных окружения
Для работы ИИ-инженера (Gemini) вам понадобится API-ключ. Создайте в корне проекта файл `.env` на основе примера `.env.example`:
```bash
cp .env.example .env
```
Откройте файл `.env` и вставьте ваш API-ключ Gemini:
```env
GEMINI_API_KEY=your_gemini_api_key_here
```
*(Получить ключ можно бесплатно в [Google AI Studio](https://aistudio.google.com/))*

### 3. Запуск в режиме разработки
Для мгновенного горячего обновления кода в процессе редактирования (HMR) запустите сервер разработки:
```bash
npm run dev
```
Сайт будет доступен по адресу: `http://localhost:3000`

## 🌐 Автоматический Деплой на GitHub Pages через GitHub Actions

Вам **не нужно** устанавливать никаких дополнительных пакетов локально или запускать команды вручную. В репозиторий уже добавлен готовый рабочий сценарий автоматического CI/CD!

Файл `.github/workflows/deploy.yml` автоматически собирает и публикует ваш сайт при каждом сохранении изменений.

### Инструкция по запуску деплоя на стороне GitHub:

1. **Создайте пустой репозиторий** на вашем аккаунте GitHub (например, с именем `sibtehlit-site`).
2. **Загрузите исходный код сайта** в этот репозиторий:
   ```bash
   git init
   git remote add origin https://github.com/ВАШ_ЛОГИН/sibtehlit-site.git
   git add .
   git commit -m "Initial commit with GitHub Actions automated deployment"
   git branch -M main
   git push -u origin main
   ```
3. **Активируйте разрешения сборщика GitHub Pages**:
   - Перейдите на страницу вашего репозитория на GitHub.
   - Зайдите в меню **Settings** (Настройки) -> **Pages** (в левой колонке).
   - В разделе **Build and deployment** (Сборка и деплой) найдите выпадающий список **Source** (Источник).
   - Выберите значение **GitHub Actions** вместо *Deploy from a branch*.
4. **Готово!** Как только вы переключите источник в статус *GitHub Actions*, автоматически запустится сборка вашего проекта. Вы можете наблюдать за процессом во вкладке **Actions**. Ссылка на ваш рабочий сайт появится там же в течение 1–2 минут!

## 🐋 Деплой с бэкендом (с поддержкой полнофункционального ИИ)

Если вы решите захостить сайт на сервере с бэкендом, у вас есть несколько вариантов:

### Вариант А. Docker (на VPS, Cloud Run, Amvera)
Готовый производительный файл `Dockerfile` и настройки `.dockerignore` уже созданы в корне вашего проекта. На сервере достаточно выполнить:
```bash
# Сборка образа
docker build -t sibtehlit-site .

# Запуск контейнера с пробросом порта 3000 и передачей API-ключа
docker run -d -p 3000:3000 -e GEMINI_API_KEY="ВАШ_КЛЮЧ" --name sibtehlit sibtehlit-site
```

### Вариант Б. Облачные хостинги (Render, Railway, Heroku)
1. Подключите ваш GitHub репозиторий к платформе (например, Railway или Render).
2. Настройки:
   - **Environment:** `Node`
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm run start`
3. Добавьте переменную окружения `GEMINI_API_KEY` в панели хостинга для живого чата.

### Вариант В. Локальный запуск на Windows 11 (WSL2 + Docker Desktop + VS Code) с доступом по локальной сети (LAN)

Этот вариант идеально подходит для локальной разработки в VS Code на Windows 11 через подсистему WSL2 с возможностью тестировать сайт со смартфонов, планшетов и других компьютеров в вашей Wi-Fi / Ethernet сети.

#### Шаг 1. Открытие проекта в VS Code под WSL
1. Убедитесь, что у вас установлен **Docker Desktop** (с включенной интеграцией WSL2 в настройках) и расширение **WSL** (от Microsoft) в VS Code.
2. Откройте терминал WSL (например, Ubuntu) и склонируйте / переместите туда папку проекта.
3. В терминале WSL перейдите в директорию проекта и запустите VS Code:
   ```bash
   cd ~/sibtehlit-site
   code .
   ```
   *VS Code запустится в режиме удаленного подключения (WSL: Ubuntu) с полной поддержкой Linux-команд.*

#### Шаг 2. Подготовка и сборка Docker-контейнера
1. Создайте в корне проекта файл `.env` (если еще не создали) и пропишите ваш ключ Gemini:
   ```env
   GEMINI_API_KEY=AIzaSy...ваш_ключ
   ```
2. Откройте встроенный терминал VS Code (Ctrl+`) и соберите Docker-образ:
   ```bash
   docker build -t sibtehlit-site .
   ```
3. Запустите контейнер:
   ```bash
   docker run -d -p 3000:3000 --env-file .env --name sibtehlit sibtehlit-site
   ```
   *Параметр `--env-file .env` автоматически передаст ключ ИИ внутрь контейнера.*

#### Шаг 3. Настройка доступа из Локальной Сети (LAN)

По умолчанию WSL2 работает за изолированным виртуальным NAT-адаптером, поэтому другие сетевые устройства не увидят запущенный контейнер напрямую по IP компьютера Windows. Ниже приведены **2 способа решения** этой задачи:

##### Способ 1: Зеркальный сетевой режим WSL (Самый простой и современный для Windows 11)
В Windows 11 появился режим `mirrored`, который объединяет сетевые интерфейсы Windows и WSL:
1. В операционной системе Windows (не внутри WSL / Docker!) откройте папку вашего пользователя. Путь: `C:\Users\Имя_Пользователя\` (или нажмите `Win+R` -> введите `%USERPROFILE%` -> Enter).
2. Создайте там файл с именем `.wslconfig` (если его нет) со следующим содержимым:
   ```ini
   [wsl2]
   networkingMode=mirrored
   ```
3. Сохраните файл и перезапустите WSL. Для этого откройте командную строку Windows (`cmd`) от имени администратора и выполните:
   ```cmd
   wsl --shutdown
   ```
4. Снова запустите Docker-контейнер. Теперь WSL и Docker работают на сетевых интерфейсах Windows, и сайт мгновенно виден в LAN!

##### Способ 2: Классический проброс портов (Portproxy Netsh)
Если вы не хотите включать зеркальный режим, можно пробросить порт 3000 с физического IP вашей Windows внутрь WSL:
1. Узнайте IP-адрес WSL в Linux-терминале приложения:
   ```bash
   ip -4 a show eth0 | grep -oP '(?<=inet\s)\d+(\.\d+){3}'
   ```
   *(Пусть, к примеру, будет `172.29.40.12`).*
2. Откройте PowerShell в Windows **от имени Администратора** и выполните команду перенаправления:
   ```powershell
   netsh interface portproxy add v4tov4 listenport=3000 listenaddress=0.0.0.0 connectport=3000 connectaddress=IP_АДРЕС_ВАШЕГО_WSL
   ```

#### Шаг 4. Открытие порта в Брандмауэре Windows (Важно!)
Чтобы Windows не блокировала внешние запросы с других устройств вашей сети к порту 3000:
1. Откройте PowerShell от имени Администратора в Windows и запустите команду:
   ```powershell
   New-NetFirewallRule -DisplayName "SibTehlit Web App LAN Port 3000" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 3000
   ```

#### Шаг 5. Подключение устройств
1. Узнайте локальный IP-адрес вашего компьютера под управлением Windows. Для этого откройте `cmd` в Windows и выполните `ipconfig`. Найдите ваш основной сетевой адаптер (Wi-Fi или Ethernet) и строку `IPv4-адрес` (например, `192.168.1.15`).
2. Со смартфона, планшета или ноутбука, подключенных к той же домашней / офисной сети Wi-Fi, перейдите в браузере по адресу:
   ```
   http://192.168.1.15:3000
   ```
3. Пользуйтесь полноценным сайтом с работающим ИИ-Ассистентом прямо в вашей локальной сети!
